1.2 Installation

Download Configurator, or clone the git repository from https://github.com/mmontone/configurator. Then run sudo sh ./install.sh. That installs configurator on /usr/local/lib/configurator directory, and the configurator binary on /usr/local/bin. After that, you can run configurator typing configurator at the shell.

1.4 Conventions

Hear are some coding conventions we’d like to follow

  • We do believe in documentation. Document your variables, functions, macros and classes. Besides, provide a documentation from a wider perspective. Provide diagrams and arquitecture documentation; examples and tutorials, too. Consider using an automatic documentation generator (see the bitacora package in the dependencies).
  • We don’t want functions to be shorter than the should nor longer than they should. There is no "every function should have at most ten lines of code" rule. We think that coding is like literature to a great extent. So you should strive for beauty and clarity.
